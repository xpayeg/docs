Subscription

One of the available payment tools that XPay offers is Recurring payments or Subscriptions.

If your service includes subscription based payments or any other form of recurring payments, you can use our endpoints to create Recurring payments that can be used to charge your user a fixed amount every set period of time with optional end date and the ability to cancel a subscription at anytime.

XPay Subscription endpoints allows you to:

  1. Create a recurring payment (subscription)
  2. Log all subscription bills with option to filter paid and unpaid bills
  3. Pay for subscribtion bill/s
  4. Cancel subscription

How it works

When you create a subscription payment it starts generating bills for each period from the start date depending on the current date and an optional end date.

For example, if you create a subscription with the start date of the current date and specify a subscription period of 30 days, it will generate a subscription bill with the current date as the due date and will generate another bill automatically every 30 days.

note

First subscription bill payment have to be requested manually, afterwards the user will be charged automatically every subscription period.

To create a subscription payment and start billing your user follow the following steps:

  1. Create Subscription
  2. Log Subscription bills
  3. Pay Subscription bill

Create Subscription

URLs

Testing server:

POST https://staging.xpay.app/api/subscriptions/

Live server:

POST https://community.xpay.app/api/subscriptions/

Parameters

Do not supply any parameters.

Authentication

Supply your API key in the request header

"x-api-key" : string

Request body

Properties

Property nameTypeRequiredDescriptionNotes
namestringTrueyour recurring payment name
amountfloatTrueFixed service/bill cost
start_datestringTrueWhen do you want to start billing your user?Format: "year-month-day", ex: "2020-9-1"
periodnumberTrueThe interval of which you bill your user in daysex: 10
member_idstringOptionalMembership ID of your userif you choose to to not supply an ID, one will be generated and sent with the response
community_idstringTrueID that is unique to your community
end_datestringOptionalIf you want to stop billing your user on on a certain date, you can supply an end dateex: "2021-9-1"

Example

The following example uses Axios which is a Promise based HTTP client for the browser and node.js

Axios Github repo

installing

Using npm:

$ npm install axios

Using jsDelivr CDN:

<script src="https://cdn.jsdelivr.net/npm/axios/dist/axios.min.js"></script>

Create a new subscription

axios({
  method: "post",
  url: "https://staging.xpay.app/api/subscriptions/",
  data: {
    name: "VIP Membership",
    amount: 1500,
    start_date: "2020-11-1",
    period: 1,
    member_id: "69",
    community_id: "m2J7eBK",
    end_date: "2020-11-30",
  },
  headers: {
    "x-api-key": "Cce74Y3B.J0P4tItq7hGu2ddhCB0WF5ND1eTubkpT",
  },
})
  .then((response) => {
    console.log(response.data.data);
  })
  .catch((e) => {
    console.log(e.response.data.status);
  });

log output

{
  "id": 1593,
  "name": "VIP Membership",
  "amount": "1500.00",
  "start_date": "2020-11-01",
  "end_date": "2020-11-30",
  "period": 1,
  "community_id": "m2J7eBK",
  "member_id": "69"
}
note

If you try to run the above code in your client you will get an error saying that the member id is used before, this is because subscriptions are linked to membership ids and the same community can't have duplicate membership ids.

Try changing the value of the member_id randomly till it work.

Log Subscription bills

This endpoint returns all the bills generated for a subscription in the form of an array of bill object

URLs

Testing server:

GET https://staging.xpay.app/api/subscriptions/bills

Production(Live) server:

GET https://community.xpay.app/api/subscriptions/bills

Parameters

Query parameters

Property nameTypeRequiredDescriptionNotes
member_idstringTrueMembership ID of your userit is the same id you supplied while creating a subscription or the one that was created automatically for you
community_idstringTrueID that is unique to your community
paidstringOptionalFilter the result by paid / unpaid subscription billsTakes values of False, false, True, true, 1, 0, yes, no

Authentication

Supply your API key in the request header

"x-api-key" : string

Example

The following example uses Axios which is a Promise based HTTP client for the browser and node.js

Pay a Subscription bill

URLs

Testing server:

POST https://staging.xpay.app/api/v1/payments/pay/bills

Production(Live) server:

POST https://community.xpay.app/api/v1/payments/pay/bills

Parameters

Do not supply any parameters.

Authentication

Supply your API key in the request header

"x-api-key" : string

Request body

Properties

Property nameValueDescriptionNotes
community_idstringID that is uniques to your community
currencystringCurrencyOnly 'EGP' is supported currently
pay_usingstringuser payment methodOnly card payments are supported
billing_data.namestringuser first and last nameMust contain first and last name in english letters with space between them.
billing_data.emailstringuser email
billing_data.phone_numberstringuser phone numbermust contain country code prefixed
amountfloatTotal Amount of bills to payMake sure to calculate the sum of the grand_total_amount value of the bills supplied in the ids parameter
idsstring[number]Ids of the bills to payYou can supply only one id in case of first payment
membership_idMembership ID of your userIt is the same id you supplied while creating a subscription or the one that was created automatically for you

Example

the following example uses Axios which is a Promise based HTTP client for the browser and node.js

Axios Github repo

installing

Using npm:

$ npm install axios

Using jsDelivr CDN:

<script src="https://cdn.jsdelivr.net/npm/axios/dist/axios.min.js"></script>

Pay a subscription bill

axios({
  method: "post",
  url: "https://staging.xpay.app/api/v1/payments/pay/bills",
  data: {
    billing_data: {
      name: "John doe",
      email: "j.doe@test.com",
      phone_number: "+201111111111",
    },
    amount: 1500,
    currency: "EGP",
    community_id: "m2J7eBK",
    pay_using: "card",
    ids: [1766],
    membership_id: "69",
  },
  headers: {
    "x-api-key": "Cce74Y3B.J0P4tItq7hGu2ddhCB0WF5ND1eTubkpT",
  },
})
  .then((data) => {
    console.log(data.data.data);
    const iframeUrl = data.data.data.iframe_url;
  })
  .catch((e) => {
    console.log(e.response.data.status);
  });

Cancel Subscription

Wether you want to stop billing the user or the user wishes to end his subscription to your services, you can easily cancel a subscription using cancel subscription endpoint

When you cancel a subscription, Xpay system will stop generating new bills for this subscription and stop billing the user for any future bills that may have been already generated.

URLs

Testing server:

POST https://staging.xpay.app/api/subscriptions/{{member_id}}/cancel

Production(Live) server:

POST https://community.xpay.app/api/subscriptions/{{member_id}}/cancel

Parameters

Property nameValueDescriptionNotes
subscription_idstringSubscription ID that you wish to cancel

Authentication

Supply your API key in the request header

"x-api-key" : string

Request body

Properties

Property nameTypeRequiredDescriptionNotes
community_idstringTrueID that is unique to your community

Example

The following example uses Axios which is a Promise based HTTP client for the browser and node.js

Axios Github repo

installing

Using npm:

$ npm install axios

Using jsDelivr CDN:

<script src="https://cdn.jsdelivr.net/npm/axios/dist/axios.min.js"></script>

Create a new subscription

const subscription_id = "put your subscription id here";
axios({
  method: "post",
  url: `https://staging.xpay.app/api/subscriptions/${subscription_id}/cancel/`,
  data: {
    community_id: "m2J7eBK",
  },
  headers: {
    "x-api-key": "Cce74Y3B.J0P4tItq7hGu2ddhCB0WF5ND1eTubkpT",
  },
})
  .then((response) => {
    if (response.status == 200) {
      console.log("Subscription Canceled!");
    }
  })
  .catch((e) => {
    console.log(e.response.data.status);
  });
